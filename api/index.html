API Reference

Integrate Buildly into your workflow with our comprehensive REST API

REST API OAuth2 JSON Rate Limited

Quick Start

The Buildly API uses OAuth2 for authentication and returns JSON responses. All API requests should be made over HTTPS.

Base URL: https://api.buildly.io/v1/
Authentication: OAuth2 Bearer Token
Rate Limit: 1000 requests per hour

Authentication

curl -X POST https://api.buildly.io/auth/token \
  -H "Content-Type: application/json" \
  -d '{
    "grant_type": "client_credentials",
    "client_id": "your_client_id",
    "client_secret": "your_client_secret"
  }'

Example Request

curl -X GET https://api.buildly.io/v1/products \
  -H "Authorization: Bearer your_access_token" \
  -H "Content-Type: application/json"

Products API

Manage your product portfolio with CRUD operations for products and their metadata.

GET /products

Retrieve all products in your portfolio

{
  "products": [
    {
      "id": "prod_123",
      "name": "My Product",
      "description": "Product description",
      "vision": "Product vision statement",
      "complexity_score": 85,
      "priority": "high",
      "created_at": "2024-01-15T10:30:00Z",
      "updated_at": "2024-01-16T14:20:00Z"
    }
  ],
  "total": 1,
  "page": 1,
  "per_page": 20
}
POST /products

Create a new product

Request Body: 
{
  "name": "New Product",
  "description": "Product description",
  "vision": "Vision statement",
  "complexity_score": 75,
  "priority": "medium"
}
GET /products/{id}

Retrieve a specific product by ID

PUT /products/{id}

Update an existing product

DELETE /products/{id}

Delete a product

AI Assistant API

Interact with BabbleBeaver AI for intelligent suggestions.

POST /ai/suggestions

Get AI suggestions for features or planning

Request Body: 
{
  "context": "product_planning",
  "product_id": "prod_123",
  "query": "What features should I prioritize for mobile app?"
}
Response: 
{
  "suggestions": [
    {
      "type": "feature",
      "title": "Offline Mode",
      "description": "Enable app usage without internet connection",
      "priority": "high",
      "reasoning": "Mobile users often have unreliable connectivity"
    }
  ],
  "confidence": 0.85
}

Error Codes

Code Status Description
400 Bad Request Invalid request parameters or body
401 Unauthorized Invalid or missing authentication token
403 Forbidden Insufficient permissions for the resource
404 Not Found Resource does not exist
429 Rate Limited Too many requests, retry after rate limit resets
500 Server Error Internal server error occurred
